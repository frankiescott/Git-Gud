git gud

This tutorial will outline the necessary steps needed in order to contribute to the club project. It will be assumed that you already have registered for a GitHub account and have Git installed on your machine. If you need any help with this, contact John or Frankie.

getting started

You will not be able to make changes to the repository master branch directly. We will be using a feature on GitHub called forking. When you fork a repository, a copy is made to your account. Navigate to the project repository. You will see a button in the top right corner of the repository page to fork.

Congratulations! You now have your own copy of the project repository.

git clone

GitHub is only a remote hosting service for git repositories. While it is entirely possible to develop within GitHub itself, you will most likely want to use your favorite text editor or IDE for development.

Navigate to your forked copy of the repository by going to your profile and clicking on the repositories tab.

Once you are on the repository page, click the green "Clone or download" button and copy the URL provided.

Open up the command line interface and navigate to the directory you would like to store the repository in. Once you have done so, issue the following command:

git clone INSERT_URL

This will clone the remote repository from GitHub and create a local copy on your machine.

git add

The add command is used to stage files for a commit. Any time you add, delete, or modify a file, the adjustment must be added to the index before a commit can be made. There are three different commands that can be used:

git add filename
git add -A
git add .
git add -u

git add filename stages individual files one at a time. git add -A stages all files in the working directory. git add . stages new and modified files, but not deleted files. git add -u stages modified and deleted files, but not new files.

git commit

The commit command takes all of the staged files that you added and records their changes into your repository.
