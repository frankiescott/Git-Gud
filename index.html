Creating A GitHub Account

Head over to the sign up page to create a GitHub account. Create a username, password, and enter your e-mail. Choose the option for unlimited free public repositories as your plan. A paid subscription is not required. You will then be asked to provide some information about your programming experience and what you plan to use GitHub for. You can skip this step or fill out the form. Congratulations! You now have a GitHub account.

Installing Git

GitHub is a only a web-based hosting service for Git repositories. It allows you to store remote copies of your repositories as backups and enables you to share and view repositories. Git is the version control system that GitHub was built on top of. GitHub offers the same functionality but Git can be integrated into a text editor or IDE as a plugin and offer more effeciency in work flow.

gitignore

A gitignore file will determine which files and directories to ignore when making a commit.

from passwords import GOOGLE_API_KEY
def google(str):
    service = build("customsearch", "v1", developerKey=GOOGLE_API_KEY)
    res = service.cse().list(q=str, cx='016654653224191036952:bj0zhyarxza',).execute()
    return res['items'][0]['link']

Notice how I import the variable "GOOGLE_API_KEY" from a file called "passwords.py" I have this file included in gitignore so it cannot be committed and pushed to the repository for everyone to see.

Another example if you are familiar with Python are the "__pycache__" folders that are created when you import a non-standard module. The files within these folders serve no purpose in the repository and can be ignored.
